The Razorpay Request Lifecycle

Phase 01: Initiation

Order Generation

Frontend triggers a POST to /api/deposit/initiate/. Django backend uses the Razorpay SDK to create a server-side order, returning the Order ID statelessly.

Phase 02: Verification

HMAC Signature

Upon successful client-side payment, Razorpay sends a payment_id and signature back to our callback node for HMAC-SHA256 validation.

Phase 03: Finalization

Atomic Update

Verified signals trigger a transaction.atomic() block, ensuring account balances and ledger entries are updated as a single, immutable unit.
